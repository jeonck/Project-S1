# 환경 설정 가이드

## 로컬 개발 환경 설정

### 1. 개발 서버 실행

```bash
npm install
npm run dev
```

**참고:** 입찰정보 페이지는 조달청 공공입찰정보 API를 사용하며, API 키는 소스코드에 직접 포함되어 있습니다.

## GitHub Pages 배포 설정

### 자동 배포

- `main` 브랜치에 푸시하면 자동으로 빌드 및 배포됩니다
- GitHub Actions 탭에서 배포 상태를 확인할 수 있습니다
- 별도의 환경변수 설정이 필요하지 않습니다

## API 정보

### 조달청 공공입찰정보서비스

- **서비스명:** 공공입찰정보서비스
- **엔드포인트:** `/getBidPblancListInfoServc`
- **제공:** 조달청 (https://apis.data.go.kr)
- **인증키:** 소스코드에 포함
- **공식 문서:** [조달청_나라장터 입찰공고정보서비스](https://www.data.go.kr/data/15129394/openapi.do)

### 주요 기능

- 최근 30일 입찰정보 조회
- 10개 컬럼 테이블 형식
  - No (순번)
  - 업무구분
  - 구분
  - 입찰공고번호 (링크 포함)
  - 공고명
  - 공고기관
  - 수요기관
  - 게시일시(입찰마감일시)
  - 단계 (입찰예정/입찰중/마감/개찰완료)
  - 입찰진행요약 (D-일수)

## 트러블슈팅

### API 응답이 없는 경우

1. 네트워크 연결 확인
2. API 서버 상태 확인
3. 브라우저 콘솔에서 에러 메시지 확인

### CORS 에러가 발생하는 경우

- 조달청 API는 클라이언트에서 직접 호출 가능합니다
- CORS 정책에 따라 일부 브라우저에서는 제한될 수 있습니다
- 필요시 프록시 서버 설정을 고려하세요
